REST Web Service

Constraints

  1. Client Server
  2. Stateless
  3. Cacheable
  4. Layered System
  5. Consistent & Uniform Interface

Alternatives

  1. Asynchronous APIs: WebSockets, MQTT, AMQP, Stomp, pubsubhubbub
  2. Orchestration APIs
  3. SDKs vs. APIs
  4. Binary protocols: Apache Thrift, Google Protocol Buffers, Apache Avro.

Advantages

  1. Simplicity
  2. Independency
  3. Scalability
  4. Encapsulation

HTTP Verbs

  1. POST: create a new resource
  2. PUT: update/replace a resource
  3. PATCH: partially update a resource
  4. DELETE: delete a resource or collection
  5. HEAD: read meta-data
  6. OPTIONS: retrieve available actions to a resource or collection

HTTP Status

Success Responses:

  1. 200: OK!
  2. 201: Resource Created
  3. 204: No Content
  4. 410: Gone
  5. ...

Error Responses:

  1. 400: Bad Request
  2. 401: Not Authorized
  3. 403: Forbidden
  4. 404: Resource Not Found
  5. 422: Unprocessable Entity
  6. 500: Internal Server Error
  7. ...

Gotchas

  1. Versioning
  2. HTTP status codes
  3. Standarization Format URIs & Payload: pagination, relations, error, etc.
  4. PUT vs. PATCH vs. POST
  5. Allow Override Method?
  6. Caching: ETag vs. Last-Modified
  7. HATEOAS?
  8. Contract/Mock & Documentation

OAuth2 Terminology

  • Authorization code

  • Client

  • Scope

Oauth2 Protocol Flow

     +--------+                               +---------------+
     |        |--(A)- Authorization Request ->|   Resource    |
     |        |                               |     Owner     |
     |        |<-(B)-- Authorization Grant ---|               |
     |        |                               +---------------+
     |        |
     |        |                               +---------------+
     |        |--(C)-- Authorization Grant -->| Authorization |
     | Client |                               |     Server    |
     |        |<-(D)----- Access Token -------|               |
     |        |                               +---------------+
     |        |
     |        |                               +---------------+
     |        |--(E)----- Access Token ------>|    Resource   |
     |        |                               |     Server    |
     |        |<-(F)--- Protected Resource ---|               |
     +--------+                               +---------------+
